# APEX Skill Bundle — MaxView

This directory contains the `.cursor/` skill and rules files that enable the
APEX end-to-end implementation flow in the MaxView repository.

## Contents

```
.cursor/
  skills/
    apex-implement-feature/
      SKILL.md          — the apex-implement-feature skill (plan + deliver)
  rules/
    coding-standards.mdc          — React/TypeScript component conventions
    ui-design-standards.mdc       — MWx Design System + CSS token usage
    existing-code-protection.mdc  — scope discipline (never touch shell/nav/header)
    testing-standards.mdc         — test file locations, mocking, coverage scope
    plan-then-implement.mdc       — two-phase workflow enforcement
    git-pr-conventions.mdc        — branch naming, commit format, PR discipline
    typescript-typecheck.mdc      — tsc --noEmit after every change
```

## Installation

Copy the `.cursor/` directory into the root of the MaxView repository and commit. The
APEX app's "Start Implementation" button will then deep-link into Cursor with
`/apex-implement-feature <adoFeatureId>` pre-filled, and the skill + rules above
will govern the entire implementation flow.

## How it works

1. The APEX app detects that a design doc is approved.
2. A "Start Implementation in Cursor" button appears on the design-doc review screen
   (visible to Admin, or the design-doc owner who is in the Developer group).
3. Clicking opens Cursor with the prompt `/apex-implement-feature <featureAdoId>`.
4. The skill reads all context from the ADO Feature (design doc attachments + acceptance
   criteria on children), drafts a Principal-Engineer implementation plan, and stops for
   human approval.
5. On approval: branch → code → tests → self-review → commit → push → PR → ADO state
   transitions ("In Progress" → "In Pull Request").
